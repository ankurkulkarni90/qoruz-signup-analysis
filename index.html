What I Found
Google OAuth signup is completely broken. 100% of users who click "Sign up with Google" hit a silent dead end. The frontend never sends a reCAPTCHA token, the backend returns a 422, and no error is shown. Tested 4 times across 2 accounts and 2 browser modes. Identical failure every time.
The manual form rejects users after they've invested effort. Users fill three fields before learning public email domains aren't accepted. Whether this is a UX gap or a deliberate lead capture strategy via the CSM modal depends on intent only the Qoruz team can clarify.
The OAuth path currently serves neither goal. It doesn't convert users (broken), doesn't capture leads (no modal shown), and doesn't provide feedback (silent failure). Fixing it is the highest-leverage improvement regardless of design philosophy.

Bug Report

Google OAuth Signup: Silent Failure

The Google OAuth signup path silently fails for all users. Root cause: the OAuth callback never generates a reCAPTCHA token, causing the registration API to return a 422. No error is shown to the user.
